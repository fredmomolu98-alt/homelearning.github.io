// Phase 2

How to Edit Your Website Content

You can personalize text, colors, and details using any free text editor — no special software needed.

3
Install a free text editor
This lets you open and edit your HTML files easily

Download and install one of these free editors — VS Code is recommended:

⬇ Download VS Code (Free) ⬇ Notepad++ (Windows)
✅ VS Code works on Windows, Mac, and Linux. It's free forever and used by millions of web developers worldwide.
4
Open and edit your files in VS Code
Change text, colors, and content to personalize your site

To open your project:

  1. Open VS Code
  2. Click File → Open Folder
  3. Select your home-learning-website folder
  4. Click on index.html in the left sidebar to open it

Common things you'll want to edit:

📝 Change the site title — Search for <title> and update the name
index.html
<!-- Find this line and change the title --> <title>Home Learning — Education for Every Student</title> <!-- Change to your own title, e.g.: --> <title>Your School Name — Home Learning</title>
🎨 Change the hero headline — Find the main heading text in the hero section
index.html — Hero Section
<!-- Find the hero title and update it --> <h1 class="hero-title"> Where Every Student<br> <span class="highlight">Learns & Grows</span> </h1> <!-- Change to your message, e.g.: --> <h1 class="hero-title"> Quality Education<br> <span class="highlight">Right at Home</span> </h1>
🌈 Change the main color — Find --amber in the CSS variables at the top
index.html — CSS Variables (top of file)
:root { /* Change --amber to any color you like */ --amber: #e8a020; /* current gold/amber */ --amber: #16a34a; /* change to green */ --amber: #2563eb; /* change to blue */ }
💡 Use Ctrl+F (Windows) or Cmd+F (Mac) in VS Code to search for any text you want to change. It's the fastest way to find what you're looking for.
5
Preview your changes in a browser
Always check how your edits look before publishing
  1. After making changes, save the file: Ctrl+S (Windows) or Cmd+S (Mac)
  2. Find the file in your folder
  3. Double-click index.html — it opens in your browser
  4. Review your changes. Go back to VS Code to adjust if needed
  5. Repeat until you're happy with the result
✅ You can install the Live Server extension in VS Code — it automatically refreshes your browser every time you save. Search "Live Server" in VS Code's Extensions panel.

Create a Free GitHub Account

GitHub is where your website will be stored and hosted — completely free.

6
Sign up for GitHub
Takes less than 2 minutes — no credit card needed
🐙 Create Free GitHub Account
  1. Go to github.com/signup
  2. Enter your email address and create a password
  3. Choose a username — this becomes part of your website URL
    e.g. username homelearning → site will be at homelearning.github.io
  4. Verify your email address
  5. Select the Free plan when asked
⚠️ Choose your username carefully — it becomes part of your public website address. Keep it short, professional, and related to your project (e.g. homelearning or hlplatform).
// Phase 4

Create a Repository & Upload Files

A "repository" is just GitHub's word for a project folder. This is where your website files live.

7
Create a new repository
This is your website's home on GitHub
  1. Log into GitHub and click the green "New" button (top left)
  2. For Repository name, type exactly: 
    YOUR-USERNAME.github.io
    Replace YOUR-USERNAME with your actual GitHub username. This exact format is required for GitHub Pages.
  3. Set visibility to Public
  4. Check "Add a README file"
  5. Click "Create repository"
✅ Example: If your username is homelearning, your repository name must be homelearning.github.io and your site will live at https://homelearning.github.io
8
Upload your website files
Drag and drop — no technical commands needed
  1. Inside your new repository, click "Add file" → "Upload files"
  2. Open your home-learning-website folder on your computer
  3. Select both files and drag them into the GitHub upload area:
    • index.html
    • home-learning-features.html
  4. Scroll down to "Commit changes"
  5. In the message box type: Add Home Learning website files
  6. Click the green "Commit changes" button
💡 A "commit" is just GitHub's way of saving your files. Think of it like pressing Save — except it also keeps a full history of every change you make.
// Phase 5

Enable GitHub Pages

One simple setting turns your repository into a live public website.

9
Turn on GitHub Pages
Your site goes live in under 60 seconds
  1. In your repository, click the ⚙ Settings tab (top menu)
  2. In the left sidebar, click "Pages"
  3. Under "Source", click the dropdown and select "Deploy from a branch"
  4. Under "Branch", select main and keep the folder as / (root)
  5. Click Save
🚀 After saving, GitHub will show a banner at the top of the Pages settings saying your site is being deployed. Refresh after 60 seconds and you'll see your live URL!
⏱ It can take 1–3 minutes for your site to go live the first time. Subsequent updates are usually faster — under 30 seconds.

11
How to update your site in the future
Making changes is just as simple as the first upload
  1. Edit your file in VS Code and save it
  2. Go to your GitHub repository
  3. Click on the file you want to update (e.g. index.html)
  4. Click the pencil ✏ icon to edit, OR click "Add file → Upload files" to replace it
  5. Upload your updated file and commit the changes
  6. Wait ~30 seconds, then refresh your live site — changes appear automatically!
💡 Bookmark your repository page at github.com/YOUR-USERNAME/YOUR-USERNAME.github.io so you can always get back to it quickly to make updates.

Editing Cheat Sheet

The most common things you'll want to edit and exactly where to find them.

📝 Text Changes
  • Site title → search <title>
  • Hero headline → search hero-title
  • Description text → search hero-sub
  • Subject names → search subject card text
  • Footer text → search <footer>
🎨 Style Changes
  • Main color → change --amber value
  • Background → change --cream value
  • Text color → change --ink value
  • Font → find fonts.googleapis.com
  • Logo text → search nav-logo-text
⌨️ VS Code Shortcuts
  • Ctrl+F → Find text
  • Ctrl+H → Find & Replace
  • Ctrl+S → Save file
  • Ctrl+Z → Undo change
  • Alt+Z → Toggle word wrap
